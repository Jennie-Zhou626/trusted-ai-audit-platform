# 现场录入数据清单

这份清单用于课堂现场从空平台开始录入。建议先点击网页右上角“清空数据”，然后按下面顺序操作。

## 1. 登记参与机构

页面：机构角色

| 机构名称 | 角色 | 钱包地址 | 联系/备注 |
|---|---|---|---|
| 机构A 数据治理中心 | 数据提供方 | `0x1000000000000000000000000000000000000001` | 负责数据来源与授权说明 |
| 机构B 联合实验室 | 数据提供方 | `0x2000000000000000000000000000000000000002` | 负责补充训练数据片段 |
| 联合训练服务方 | 训练方 | `0x3000000000000000000000000000000000000003` | 负责训练任务与模型版本登记 |
| 第三方审计方 | 审计方 | `0x4000000000000000000000000000000000000004` | 负责哈希复算与证据链复核 |

## 2. 创建项目

页面：项目

| 字段 | 建议输入 |
|---|---|
| 项目名称 | 多机构 AI 模型训练过程可信审计项目 |
| 说明 | 机构A与机构B共同提供训练数据，训练方登记模型版本，审计方验证训练过程证据链。 |

创建后，项目 ID 通常是 `1`。

## 3. 登记数据集 A

页面：数据集

| 字段 | 建议输入 |
|---|---|
| 数据集名称 | 机构A训练数据片段 |
| 提供机构 | 机构A 数据治理中心 |
| 来源说明 | 机构A授权提供的模型训练数据片段 |
| 授权类型 | 科研使用 |
| 数据文件 | `examples/showcase/data/iris_org_a.csv` |

登记后，数据集 ID 通常是 `1`。

## 4. 登记数据集 B

页面：数据集

| 字段 | 建议输入 |
|---|---|
| 数据集名称 | 机构B补充数据片段 |
| 提供机构 | 机构B 联合实验室 |
| 来源说明 | 机构B授权提供的补充训练数据片段 |
| 授权类型 | 科研使用 |
| 数据文件 | `examples/showcase/data/iris_org_b.csv` |

登记后，数据集 ID 通常是 `2`。

## 5. 登记训练任务

页面：训练任务

| 字段 | 建议输入 |
|---|---|
| 任务名称 | 联合训练任务 v1 |
| 数据集 ID 列表 | `[1, 2]` |
| 算法 | `FederatedAvg(LogisticRegression)` |
| 任务说明 | 使用机构A和机构B登记数据训练轻量分类模型，平台只审计材料哈希与登记关系。 |
| 训练代码 | `examples/showcase/code/train_iris.py` |
| 参数配置 | `examples/showcase/config/iris_config.json` |

登记后，训练任务 ID 通常是 `1`。

## 6. 登记协同训练轮次

页面：协同轮次

| 字段 | 第一条建议输入 | 第二条建议输入 |
|---|---|---|
| 训练任务 ID | `1` | `1` |
| 轮次 | `1` | `1` |
| 参与机构 | 机构A 数据治理中心 | 机构B 联合实验室 |
| 本地 Epoch | `2` | `2` |
| 样本量 | `75` | `75` |
| 梯度/参数哈希 | 留空自动生成 | 留空自动生成 |
| 检查点 URI | `ipfs://demo-iris-round-1-org-a` | `ipfs://demo-iris-round-1-org-b` |
| 隐私机制 | 联邦学习摘要 | 联邦学习摘要 |

可再补一条聚合轮次：

| 字段 | 建议输入 |
|---|---|
| 训练任务 ID | `1` |
| 轮次 | `2` |
| 参与机构 | 联合训练服务方 |
| 本地 Epoch | `1` |
| 样本量 | `150` |
| 检查点 URI | `ipfs://demo-iris-global-checkpoint-v1` |
| 隐私机制 | 仅哈希存证 |

## 7. 登记模型版本

页面：模型版本

| 字段 | 建议输入 |
|---|---|
| 模型名称 | 分类模型 v1 |
| 训练任务 ID | `1` |
| 指标 JSON | `{"accuracy":0.9667,"f1_score":0.9615}` |
| 模型文件 | `examples/showcase/models/iris_model_v1.pkl` |

登记后，模型版本 ID 通常是 `1`。

## 8. 发起审计

页面：审计验证

| 字段 | 建议输入 |
|---|---|
| 模型版本 ID | `1` |
| 审计说明 | 自动哈希一致性审计 |

预期结果：审计记录显示 `passed`。

## 9. 查询证据链

页面：证据链

| 字段 | 建议输入 |
|---|---|
| 模型版本 ID | `1` |

预期结果：

```text
数据集 #1 / #2
  -> 训练任务 #1
  -> 协同训练轮次 #1 / #2 / #3
  -> 模型版本 #1
  -> 审计记录 passed
```

## 10. 生成审计报告

页面：审计报告

| 字段 | 建议输入 |
|---|---|
| 模型版本 ID | `1` |

预期结果：显示可信评分、优势、风险与改进建议。登记了协同轮次且审计通过时，可信评分应较高。

## 11. 可选：说明篡改检测

页面：审计验证

1. 输入模型版本 ID：`1`。
2. 点击“篡改当前模型文件”。
3. 再次发起审计。

预期结果：新增审计记录显示 `failed`，失败原因是模型文件哈希不一致。
