# 汇报讲解路线

本文档用于配合 PPT 说明平台原型，不绑定某一个固定项目。网页中已有的数据可以作为运行状态示例；如果数据被清空，也可以按正常流程重新创建项目、登记材料并发起审计。

## 启动入口

前端：

```text
http://127.0.0.1:5173
```

后端健康检查：

```text
http://127.0.0.1:8000/api/health
```

链状态：

```text
http://127.0.0.1:8000/api/chain/status
```

## 推荐讲解顺序

### 1. 总览页

说明平台已经把训练过程拆成五类对象：

- 项目
- 数据集
- 训练任务
- 模型版本
- 审计记录

重点强调：平台主能力是训练过程证据审计，不是完整 AI 训练平台。

### 2. 项目页

展示平台以“项目”为上层容器组织审计对象。一个项目可以对应一次多机构模型研发、一次课程实验或一次模型版本审计任务。

### 3. 数据集页

说明数据文件保存在链下，平台登记：

- 数据来源
- 提供机构
- 授权类型
- 数据文件 SHA-256 哈希
- 链上交易哈希或模拟交易哈希

讲解点：不把隐私数据或大文件直接放上链，只把摘要和关键元数据作为证据锚点。

### 4. 训练任务页

说明训练任务用于连接数据、代码和配置。平台会记录：

- 使用的数据集 ID
- 训练算法
- 训练代码哈希
- 参数配置哈希
- 登记交易哈希

讲解点：模型版本不是孤立登记，而是能够追溯到具体训练任务。

### 5. 模型版本页

说明模型文件也保存在链下，平台记录模型文件哈希和指标 JSON。审计时重新计算当前模型文件哈希，与登记哈希比对。

### 6. 审计验证页

说明审计逻辑：

```text
重新计算链下文件哈希
  -> 比对登记哈希
  -> 检查数据集授权字段
  -> 输出 passed / failed
  -> 登记审计记录
```

如果需要说明异常情况，可以使用“链下材料篡改检测”按钮对模型文件追加测试内容，然后再次审计，观察模型哈希不一致导致审计失败。

### 7. 证据链页

输入模型版本 ID，查看完整追溯链：

```text
数据集 -> 训练任务 -> 模型版本 -> 审计记录
```

讲解点：平台不是只记录最终模型，而是把训练材料、训练任务、模型版本和审计结论组织成可复核的证据链。

## 和 PPT 的配合

PPT 中先讲问题背景、链上链下协同机制和平台架构；网页平台负责补充“系统已经如何落地”。建议把网页作为运行原型来讲，不必把某个样例项目作为唯一场景。

## 一句话总结

本平台通过链下保存训练材料、链上登记关键哈希和元数据，将多机构 AI 模型开发过程组织为可追溯、可验证、可复核的证据链。
