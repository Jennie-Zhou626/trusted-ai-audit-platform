# Trusted AI Audit Platform

多机构 AI 模型训练过程可信审计平台。

## 平台边界

平台主能力是训练过程审计，不是完整 AI 训练平台。训练模块只作为平台接入对象和验证用例存在。

## 模块

- `apps/web`：前端网页。
- `apps/api`：后端 API。
- `contracts`：Solidity 智能合约。
- `adapters`：训练适配器。
- `storage`：链下文件存储。
- `docs`：工程架构说明。
- `scripts`：启动、部署、自测和数据准备脚本。

## 第一阶段目标

先完成项目、数据集、训练任务、模型版本和审计记录的本地数据闭环，再逐步接入哈希、区块链和审计验证。

## 当前实现状态

已完成第一版可运行骨架：

- React 网页：总览、项目、数据集、训练任务、模型版本、审计验证、证据链页面。
- FastAPI 后端：项目、数据集、训练任务、模型版本、审计记录 API。
- 机构角色：支持登记数据提供方、训练方、审计方和监管/观察方。
- 角色权限控制：前端支持选择当前角色，并按数据提供方、训练方、审计方、监管方限制可执行操作。
- 协同训练轮次：支持记录多机构训练轮次、梯度/参数哈希、检查点 URI 和隐私机制说明。
- SQLite 数据库：自动初始化业务表。
- 文件上传：保存链下材料并计算 SHA-256。
- 审计验证：重新计算链下文件哈希并输出逐项比对结果，覆盖数据集、训练代码、配置和模型文件。
- 审计报告：按模型版本生成可信评分、优势、风险和改进建议。
- 链上登记：默认自动检测 Hardhat 部署文件，优先使用 web3.py 调用真实合约；不可用时回退 `MockChainClient`。
- Solidity 合约草案：`contracts/contracts/AuditRegistry.sol`。
- 篡改检测：可在审计页对模型文件追加测试内容，再次审计会显示哈希不一致。
- 证据链详情：可按模型版本 ID 查询数据集、训练任务、协同轮次、模型版本和审计记录，并用流程图呈现链路。
- 报告导出：审计报告页支持导出 Markdown 和 JSON 报告。

## 启动

Windows 下建议直接使用 `.cmd` 脚本，避免 PowerShell 执行策略拦截。

后端：

```powershell
.\scripts\start-api.cmd
```

前端：

```powershell
.\scripts\start-web.cmd
```

访问：

```text
http://127.0.0.1:5173
```

API 健康检查：

```text
http://127.0.0.1:8000/api/health
```

如需恢复一组本地样例数据，可运行：

```powershell
.\scripts\seed-sample-project.cmd
```

该脚本只用于开发和自测，不是平台主流程。正常使用时可直接在网页中创建项目、登记数据集、训练任务和模型版本。

## 基本使用流程

如果希望从空平台开始，可先点击网页右上角“清空数据”。该操作会删除本地数据库记录和已上传的链下材料索引。

1. 在页面右上角选择当前角色。
2. 使用“数据提供方”维护机构、项目并登记数据集。
3. 使用“训练方”登记训练任务、协同训练轮次和模型版本。
4. 使用“审计方”发起哈希一致性审计或篡改检测。
5. 使用“监管方”只读查看总览、证据链和审计报告。

当前权限控制是前端原型，用于明确多机构协作中的操作边界。真实部署时应接入登录、机构身份认证和后端鉴权。

真实本地链：

```powershell
.\scripts\start-chain.cmd
.\scripts\deploy-contract.cmd
```

部署完成后，后端会自动检测 `contracts/deployment.json`。如果 Hardhat RPC 可用，链模式显示为 `web3`；否则回退为 `mock`。

## 自测

```powershell
.\scripts\smoke-test-api.cmd
```

自测会创建一组项目、数据集、训练任务、模型版本和审计记录，审计结果应为 `passed`。
